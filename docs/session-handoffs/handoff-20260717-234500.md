## 4. GP-518 root cause (the durable finding)

`app/api/lms/courses/[slug]/route.ts` **never existed** — `git log --all` over the path returned
empty. Requests fell through to the `[[...path]]` catch-all's 503 stub, which the DigitalOcean
edge rewrites to an HTML 504.

**The ticket's premise is wrong and will mislead anyone who reads it.** It says 504 implies a hang,
pointing at "an unbounded query or an upstream call without a timeout". There is no query and no
hang. Evidence:

- `504` returned in **0.104s** — not a timeout.
- Header `x-do-orig-status: 503`, `content-type: text/html`.
- **A nonexistent slug also returned 504** — proves route-wide, not data-driven.
- The list route's own docstring documents this identical mechanism, already fixed there once.

**Public page was never affected** — `getCourse()` reads the DB first
(`getPublishedCourseDetailBySlugFromDatabase`) and only falls back to the API when
`DATABASE_URL` is unset. Prod has it set.
