This is a research + design package, not a finished shippable course. It gives the founder enough
depth to (a) see the market gap being filled, (b) send the numbers and methodology to an accountant
for verification, and (c) greenlight the build of the pricing calculator.

## Key design decisions

- **Cost-based, not competitor-copying** — the course's entire premise: price from *your* real costs.
- **The owner's real numbers** — manual entry, import from cloud bookkeeping (reusing RestoreAssist's
  existing Xero/QuickBooks/MYOB OAuth layer; note the P&L *read* is a new capability to build), or CSV.
- **Accountant-verification gate** — an explicit step + a one-page worksheet the owner takes to their
  accountant, plus a disclaimer on course start, in Module 9, and on every calculator output.
- **Every number cited** — nothing asserted that isn't traceable to `research-and-figures.md`;
  unverified items flagged `[NEEDS CURRENT FIGURE]` / `[UNVERIFIED — accountant to confirm]`.
- **Annual maintenance** — award rates change each 1 July, tax/thresholds at budget; the course needs a
  scheduled yearly refresh of the figures and calculator defaults.
