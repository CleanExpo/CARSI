# `src/components/ui/` — shadcn drift inventory

This directory hosts components originally generated by the [shadcn/ui](https://ui.shadcn.com) CLI. Several have been modified in place to match the CARSI "Abyssal Vision" glass design system (see GP-335, GP-362). This file tracks every **intentional** divergence so future `npx shadcn@latest add <name>` runs don't silently clobber custom behaviour.

## Rules of engagement

1. **Never blindly re-run `npx shadcn@latest add <component>`** on any component listed below — it will overwrite local changes.
2. For components not listed, the CLI may be used freely; they are canonical.
3. When adding a new intentional diff, append it here in the same session as the code change.
4. Root shadcn config lives in `/components.json` (added in this ticket).

## Intentional diffs (do NOT overwrite)

### `button.tsx`
- Adds brand variants: `gradient`, `glow`, `glow-accent`, `success`, `warning`, `info`.
- Adds sizes: `xl`, `icon-sm`, `icon-lg`.
- Default variant adds `active:scale-[0.98]` micro-interaction and `transition-transform` on press.
- Uses `shadow-glow` / `shadow-glow-accent` tokens defined in `tailwind.config.ts` (see GP-362 glass tokens).

### `card.tsx`
- Exported as a CVA component with variants: `elevated`, `interactive`, `featured`, `gradient`, `glass`, `outline`, `ghost`.
- Adds `asChild` prop via `@radix-ui/react-slot` semantics so cards can wrap `<Link>`.
- Imports `next/image` for the `featured` variant header slot.
- References brand gradient tokens (`bg-gradient-brand`, `brand-primary`) from GP-335 namespace.

### `badge.tsx`
- Extended variants align with button's brand palette (`success`, `warning`, `info`, `gradient`). Keep in sync with `button.tsx` when adding new brand colours.

## Non-shadcn components co-located here

These were never shadcn-generated and must not be re-added by the CLI:

- `AcronymTooltip.tsx` — CARSI domain-specific tooltip that wraps shadcn `Tooltip` with an acronym dictionary (IICRC, WRT, ASD…).
- `loading.tsx` — CVA spinner + skeleton presets used across LMS.
- `motion.tsx` — lightweight animation wrapper (`fade-in`, `slide-up`, etc.), no Framer Motion dependency.
- `password-input.tsx` — `Input` wrapper with visibility toggle (lucide `Eye` / `EyeOff`).
